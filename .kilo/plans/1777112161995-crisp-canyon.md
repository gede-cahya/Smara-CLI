# Plan: PRD Persistent Storage / Memory untuk Smara CLI

## Status: READY FOR IMPLEMENTATION

## Goals
Membuat PRD komprehensif untuk **Persistent Storage / Memory** berdasarkan analisis mendalam terhadap sistem yang sudah ada.

---

## Assumptions (Berdasarkan analisis codebase & konteks "lanjut" dari user)

1. **Scope**: Enhancement sistem memory yang ada (SQLite-based), bukan rewrite total
2. **Target**: Team/Enterprise use dengan dukungan multi-user collaboration
3. **Storage**: Tetap gunakan SQLite sebagai primary, dengan interface untuk extensibility
4. **Search**: Hybrid approach (keyword + semantic/vector search yang sudah ada)
5. **Sync**: Dokumentasikan dan enhance existing sync mechanism
6. **Features**: Fokus pada organization, search, retention, dan team sharing

---

## PRD Structure (Final)

### I. Executive Summary
- Visi: "Memori tim yang cerdas, sinkron, dan scalable"
- Problem: Current system lacks organization, retention policies, dan advanced search
- Success Metrics: Search relevance >85%, Sync latency <5min, Zero data loss

### II. Konteks & Motivasi
- Current state: SQLite store dengan vector search (embedding)
- Pain points teridentifikasi dari codebase analysis
- User stories dari perspektif developer & team lead

### III. User Stories & Requirements

**P0 - Core Persistence:**
- US-01: Save/retrieve memory dengan metadata lengkap
- US-02: Workspace isolation (sudah ada, perlu dokumentasi)
- US-03: Vector search dengan similarity scoring (sudah ada)
- US-04: Cloud sync dengan delta tracking (sudah ada, perlu enhancement)

**P1 - Organization & Search:**
- US-05: Tag-based memory organization
- US-06: Full-text search in addition to vector search
- US-07: Memory folders/categories within workspace
- US-08: Advanced filtering (date range, source, tags)

**P2 - Management & Collaboration:**
- US-09: Retention policies (TTL, auto-cleanup)
- US-10: Memory sharing antar workspace
- US-11: Export/import (JSON, Markdown)
- US-12: Memory versioning & diff

### IV. Spesifikasi Teknis

**Data Models** (enhanced dari `internal/memory/types.go`):
```go
type Memory struct {
    ID          int64
    WorkspaceID int64
    Content     string
    Embedding   []float32
    Tags        []string  // Enhanced: parse dari string
    Category    string    // NEW
    Source      string
    Metadata    map[string]interface{} // NEW
    CreatedAt   time.Time
    UpdatedAt   time.Time // NEW
    ExpiresAt   *time.Time // NEW: TTL
}
```

**Storage Interface** (from `types.go:68-102`):
- Maintain existing interface
- Add: UpdateMemory, GetMemoryByID, SearchByMetadata
- Enhance: List with filters (tags, date range, category)

**Search Architecture**:
- Combine existing vector search (`search.go`) dengan full-text FTS5
- Hybrid scoring: 0.6 * vector_similarity + 0.4 * text_relevance
