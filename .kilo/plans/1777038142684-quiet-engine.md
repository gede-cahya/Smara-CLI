# Plan: Integrate Explore as Streaming Output (like Thinking/Thought/Worked)

## Context

User wants the `explore` feature to be displayed similarly to `Thinking`, `Thought`, and `Worked` tags - as inline/streaming output during agent processing, not just as a static response after the fact.

## Current State

- `Thinking`, `Thought`, `Worked` are displayed as inline streaming content in `app.go` during agent processing
- `Explore` currently outputs only after agent finishes (static)
- User wants explore to appear live during processing like thinking does

## Goal

Integrate explore output as a streaming inline display alongside Thinking/Thought/Worked during agent processing.

## Implementation Steps

### 1. Add ExploreCallback to AgenticCallback
In `internal/agent/types.go` or callback definition:
```go
OnExplore func(path string, results string) // Called when explore runs
```

### 2. Create Explore Streaming Message Type
In `internal/ui/app.go`:
```go
type ExploreMsg struct {
    Path    string
    Content string
}
```

### 3. Modify AppModel to Handle ExploreMsg
- Add `currentExplore string` field to store live explore output
- Handle `ExploreMsg` in `Update()` to update viewport
- Render explore content alongside thinking in `View()`

### 4. Modify Agent/Supervisor to Emit ExploreMsg
When `analyze_workspace` or `/explore` runs during agent processing, send `ExploreMsg` to update UI in real-time.

### 5. Trigger Explore from Agent
The `analyze_workspace` builtin tool already walks directories. Modify to send streaming updates to UI as it discovers files.

## File Changes

| File | Action |
|------|--------|
| `internal/ui/app.go` | Add `ExploreMsg`, `currentExplore string`, `exploreVisible bool`, handle in Update/View |
| `internal/agent/types.go` | Add `OnExplore func(path string, results string)` to `AgenticCallback` |
| `internal/agent/supervisor.go` | Call `ExploreCodebase` on each ProcessPrompt, invoke OnExplore |
| `cmd/smara/start.go` | Wire `OnExplore` callback to send `ExploreMsg` to TUI |
