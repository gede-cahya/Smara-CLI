# Smara Desktop - Product Requirements Document (PRD)

## Overview
Smara Desktop is a Wails-based desktop AI agent application with React frontend. It provides an interactive chat interface for interacting with AI models through the Smara CLI backend.

---

## Feature 1: Workspaces Management

### Description
Users can organize their work into multiple workspaces, each containing separate sessions and configurations.

### Requirements
- **Display workspace list** in sidebar from Go backend via `GetWorkspaces()`
- **Select workspace** by clicking - calls `SetWorkspace(id)` to switch active workspace
- **Visual indicator** for currently active workspace (highlighted)
- **Auto-load** sessions when workspace changes

### Data Flow
```
Go: memStore.ListWorkspaces() → App.GetWorkspaces() → Frontend
     App.SetWorkspace(id) → Go: supervisor.SetWorkspaceID(id)
```

### UI Elements
- Workspace list in sidebar
- Active workspace highlighted with border-left accent

---

## Feature 2: Session Management

### Description
Sessions store conversation history within a workspace. Users can create, switch, and resume conversations.

### Requirements
- **List sessions** for active workspace via `GetSessions()`
- **Switch session** by clicking - calls `SwitchSession(id)`
- **Create new session** via + button - calls `CreateSession(name)`
- **Session history** loaded via `GetSessionHistory()` on switch
- **Auto-create default session** if none exists

### Data Flow
```
GetSessions() → Go: sessStore.ListSessionsByWorkspace(wID)
SwitchSession(id) → Go: supervisor.SwitchSession(id)
GetSessionHistory() → Go: supervisor.GetCurrentSession().History
CreateSession(name) → Go: supervisor.CreateSession(config)
```

### UI Elements
- Session list in sidebar with + button
- Active session highlighted

---

## Feature 3: Chat Interface

### Description
Main conversation area where users send messages and receive AI responses.

### Requirements
- **Send message** - user clicks send or presses Enter
- **Display messages** - user messages (right-aligned, accent color) and AI responses (left-aligned)
- **Markdown rendering** - AI responses rendered with `marked` library (code blocks, lists, etc.)
- **Streaming response** - AI responses appear via `stream_chunk` event from Go backend
- **Thinking indicator** - show animated dots while AI is processing

### Data Flow
```
Send: Ask(text) → Go: supervisor.ProcessPrompt() → response
Stream: Go: runtime.EventsEmit("stream_chunk", data) → Frontend: EventsOn("stream_chunk", callback)
History: GetSessionHistory() → returns llm.Message[] with role: "user" | "assistant"
```

### UI Elements
- Message bubbles (user = accent color, AI = darker bg)
- Markdown content support (code blocks, bold, lists)
- Typing indicator with animated dots

## Feature 4: Tools Display

### Description
Shows all available tools from both built-in and MCP (Model Context Protocol) sources.

### Requirements
- **List all tools** via `GetTools()`
- **Display tool badges** in sidebar
- **Tool info** includes name, description, source (builtin vs MCP server name)
- **Refresh tools** after config changes

### Data Flow
```
GetTools() → Go: agent.GetBuiltinTools() + supervisor.GetMCPInfo()
```

### UI Elements
- Tool badges in sidebar with green indicator dot
- Scrollable if many tools

---

## Feature 5: Settings Modal

### Description
Configuration panel for LLM provider settings.

### Requirements
- **Open settings** - click Settings in sidebar
- **Provider selection** - dropdown: Ollama, OpenAI, OpenRouter, Anthropic, Custom
- **Model configuration** - text input for model name
- **API Key** - password input (shown/hidden)
- **Base URL** - text input for custom endpoints
- **Save changes** - calls `UpdateConfig(cfg)` and reinitializes

### Data Flow
```
GetConfig() → Go: config.Get() → returns SmaraConfig
UpdateConfig(cfg) → Go: config.Save() + app.initSmara()
```

### UI Elements
- Modal overlay with glass effect
- Form fields: Provider dropdown, Model, API Key, Base URL
- Cancel and Save buttons

---

## Feature 6: Theme Toggle

### Description
Switch between dark and light themes.

### Requirements
- **Toggle button** in header
- **Theme state** stored in React useState
- **CSS variables** change based on `data-theme` attribute
- **Persist preference** (optional - not implemented yet)

### UI Elements
- Theme toggle button with sun/moon icon

---

## Feature 7: Real-time Streaming

### Description
AI responses stream in real-time as they are generated, showing partial results.

### Requirements
- **Subscribe to stream events** via `EventsOn("stream_chunk", callback)`
- **Append chunks** to last AI message as they arrive
- **Skip thinking chunks** (`is_thinking: true`)
- **Combine streaming** with final response from `Ask()`

### Data Flow
```
Go: runtime.EventsEmit(ctx, "stream_chunk", {chunk, is_thinking})
Frontend: EventsOn("stream_chunk", handler)
```

### UI Elements
- Messages update in-place as chunks arrive
- Smooth scroll to bottom on new content

## Technical Stack

### Frontend
- **Framework**: React 18 with TypeScript
- **Build**: Vite 5 with @vitejs/plugin-react
- **Styling**: CSS with CSS variables for theming
- **Markdown**: marked + highlight.js

### Backend (Go)
- **Framework**: Wails v2
- **LLM**: Via pkg/llm provider system
- **Memory**: SQLite via pkg/memory
- **Session**: SQLite via pkg/session

### Wails Bindings
```typescript
// Auto-generated bindings (wailsjs/go/main/App.js)
Ask(prompt: string): Promise<string>
GetWorkspaces(): Promise<Workspace[]>
GetSessions(): Promise<Session[]>
SwitchSession(id: string): Promise<void>
GetSessionHistory(): Promise<Message[]>
CreateSession(name: string): Promise<string>
SetWorkspace(id: number): Promise<void>
GetConfig(): Promise<Config>
UpdateConfig(cfg: Config): Promise<void>
GetTools(): Promise<Tool[]>
```
