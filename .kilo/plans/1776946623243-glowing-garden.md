# Plan: Todos Sidebar, Code Diff, and Explore Codebase

## Overview
Add a **Todos sidebar** to the TUI that tracks agent file edits, shows **before/after code diffs** with syntax highlighting theme, and a new **explore codebase** feature.

## Files to Modify / Create

### New Files
- `internal/ui/todo.go` — Todo list data model, diff rendering, syntax theme
- `internal/ui/explore.go` — Explore codebase feature

### Modified Files
- `internal/ui/app.go` — Add sidebar viewport, todo state, View() layout, message types
- `internal/agent/builtin_tools.go` — Wire edit_file tool to broadcast diff info via supervisor callback
- `internal/agent/supervisor.go` — Add `OnEdit` callback hook; ensure `executeToolCall` emits edit events
- `cmd/smara/start.go` — Register `OnEdit` callback when TUI starts

## Step-by-Step Plan

### 1. Define Todo & Diff Data Models (`internal/ui/todo.go`)
Create a new file `internal/ui/todo.go`:

- `EditTodo` struct:
  - `ID string`
  - `FilePath string`
  - `Status string` (`pending`, `done`, `failed`)
  - `OldContent string`
  - `NewContent string`
  - `Description string` (e.g. "edit_file: main.go")
  - `CreatedAt time.Time`

- `TodoList` struct:
  - `todos []EditTodo`
  - `selected int` (currently selected todo for detail view)
  - `width int`
  - `height int`
  - Methods: `Add()`, `SetStatus()`, `Render()`, `Selected()`, `NavigateUp()`, `NavigateDown()`

- `DiffRenderer`:
  - `RenderDiff(old, new string) string` → returns a unified-diff-like string using `lipgloss` styles
  - **Syntax Theme** (lipgloss-based, no external library):
    - `diffAddedStyle` — green (`#A6E22E`) for added lines (`+`)
    - `diffRemovedStyle` — red (`#F92672`) for removed lines (`-`)
    - `diffContextStyle` — dim gray (`#75715E`) for unchanged context
    - `diffHeaderStyle` — cyan (`#66D9EF`) for file path / hunk header
    - `diffLineNumStyle` — dark gray (`#5F5F5F`) for line numbers
  - Use a simple line-by-line diff: split by `\n`, compare with basic LCS or even just highlight changed lines. Keep it **simple** to avoid heavy dependencies.
  - For code blocks inside the chat, reuse the same `codingStyle` already in `app.go` but enhance with keyword/string/comment colors via regex:
    - `keywordStyle` — magenta (`#F92672`) for Go keywords (`func`, `package`, `import`, `return`, etc.)
    - `stringStyle` — green (`#A6E22E`) for quoted strings
    - `commentStyle` — gray italic (`#75715E`) for `//` and `/* */`
    - `numberStyle` — purple (`#AE81FF`) for numeric literals
    - `typeStyle` — cyan (`#66D9EF`) for types and built-in names
  - `HighlightCode(content, lang string) string` — applies regex-based coloring.

### 2. Add Sidebar State to `AppModel` (`internal/ui/app.go`)
Modify `AppModel`:

- Add `todoList TodoList`
- Add `showSidebar bool` (toggle with `Ctrl+B` or `[`)
- Add `sidebarViewport viewport.Model`
- Add `sidebarWidth int` (default ~35% of terminal width, min 30 cols)

Add new `tea.Msg` types:
- `EditTodoMsg` — carries `EditTodo` data, sent from agent to UI
- `ToggleSidebarMsg` — toggles sidebar visibility

### 3. Update `View()` Layout (`internal/ui/app.go`)
Change layout from single-column to split when `showSidebar` is true:

```
+-----------+----------+
|  Header   |  Header  |
+-----------+----------+
|           |          |
|  Chat     |  Sidebar |
|  Viewport |  (Todos) |
|           |          |
+-----------+----------+
|  Input    |  Input   |
+-----------+----------+
```

Implementation:
- Use `lipgloss.JoinHorizontal(lipgloss.Top, leftPanel, rightPanel)` to compose the two panes.
- Left pane: existing `borderStyle.Render(m.viewport.View())`.
- Right pane: new `sidebarBorderStyle.Render(m.sidebarViewport.View())`.
- When `showSidebar` is false, keep existing single-column layout (no change).
- Update `WindowSizeMsg` handler to proportionally resize both viewports:
  - `chatWidth = totalWidth - sidebarWidth - 3` (borders)
  - Ensure `chatWidth >= 40` and `sidebarWidth >= 25`; if not, auto-hide sidebar.

### 4. Handle `EditTodoMsg` in `Update()` (`internal/ui/app.go`)
Add new `case EditTodoMsg:` block:
- `m.todoList.Add(msg.Todo)`
- Re-render sidebar content: `m.sidebarViewport.SetContent(m.todoList.Render())`
- Also inject a compact summary into chat history (e.g., "📝 Edit: main.go (12 lines changed)") via `m.addMessage("System", ...)` so the user sees it in the main log even if sidebar is hidden.

### 5. Intercept `edit_file` in Agent (`internal/agent/builtin_tools.go` + `supervisor.go`)

**`builtin_tools.go` — `executeBuiltinTool`:**
- Before writing the file, capture the **old content** (already read at line 452).
- After successful write, capture the **new content** (re-read or already computed).
- Return a structured result instead of plain string: change return type from `string` to a struct, or extend the return with old/new content.

**`supervisor.go` — `executeToolCall`:**
- After calling `executeBuiltinTool` (or any tool), check if the tool name is `edit_file`.
- If so, read the returned diff info and invoke the new callback:
  ```go
  if s.callback.OnEdit != nil {
      s.callback.OnEdit(filePath, oldContent, newContent)
  }
  ```

**`supervisor.go` — `AgenticCallback` struct:**
- Add `OnEdit func(path, old, new string)` field.

### 6. Wire Callback in `cmd/smara/start.go`
In the `supervisor.SetCallback(...)` block, add:
```go
OnEdit: func(path, old, new string) {
    p := ui.GetGlobalProgram()
    if p != nil {
        p.Send(ui.EditTodoMsg{
            Todo: ui.EditTodo{
                FilePath: path,
                OldContent: old,
                NewContent: new,
                Status: "done",
                Description: fmt.Sprintf("edit_file: %s", path),
                CreatedAt: time.Now(),
            },
        })
    }
},
```

### 7. Add `/explore` Command (`internal/ui/explore.go` + `app.go` + `start.go`)
Create `internal/ui/explore.go`:

- `ExploreResult` struct with `Path`, `Type` (dir/file), `Size`, `Language`.
- `ExploreCodebase(root string, depth int) ([]ExploreResult, error)`:
  - Walk directory tree up to `depth` (default 3).
  - Skip: `.git`, `node_modules`, `vendor`, `.kilo`, `dist`, `build`, `__pycache__`, `.next`.
  - Detect language by extension and return icon + color.
  - Return a tree-like string suitable for viewport rendering.
- `RenderExplore(results []ExploreResult) string` — tree view with `lipgloss` styling.

Add command `/explore [path] [depth]`:
- In `handleCommand()` in `app.go` and `start.go`.
- Parse args; default path `.`, default depth `2`.
- Run explore synchronously (fast operation).
- Inject rendered tree into chat as a `System` message (or into sidebar if sidebar is open).

### 8. Keyboard Shortcuts
Add to `Update()` (`KeyMsg`):
- `ctrl+b` or `[` — Toggle sidebar (`m.showSidebar = !m.showSidebar`), resize viewports.
- `↑ / ↓` in sidebar (when focused) — Navigate todos. (We can defer focus management to v2; for v1, just show latest todo details in sidebar and let `↑/↓` cycle selection.)

### 9. Styling & Polish
- Sidebar border: `sidebarBorderStyle = lipgloss.NewStyle().Border(lipgloss.NormalBorder()).BorderForeground(lipgloss.Color("#3C3C3C")).Width(...)`
- Sidebar title: "📋 Todos" rendered with `titleStyle`.
- Todo item rendering:
  - Status icon: `⏳` pending, `✅` done, `❌` failed.
  - File path in cyan bold.
  - Short diff summary (e.g., "+12 -5 lines").
- Selected todo: inverse/highlight background (`#7D56F4`).

### 10. Testing & Edge Cases
- If terminal width < 65, auto-hide sidebar and print a one-time hint.
- If `edit_file` fails (old_content not found), still emit `EditTodoMsg` with `Status: "failed"` so the user sees the attempt.
- Ensure `go build ./...` passes after all changes.

## Dependencies
No new external Go dependencies required. We only use:
- `github.com/charmbracelet/lipgloss` (already in go.mod)
- `github.com/charmbracelet/bubbles/viewport` (already in go.mod)
- Standard library (`strings`, `regexp`, `path/filepath`, `os`, `time`, etc.)
